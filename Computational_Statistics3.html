Accept-Reject¶

Dans l'énoncé, une erreur sur la gamma pour la variable aléatoire de l'Accept-Reject, puisque la loi proposée (Gamma(4,7)) avait des queues de distribution plus fine que la loi que l'on cherche (Gamma(4.3,6.2)). Nous allons donc travailler avec la loi Gamma(4,6) qui elle possède bien des queues de distribution plus épaisse que la loi que l'on cherche.

In [1]:
#Accept-Reject
iterations = 10000
alpha=4
beta=6
alphai=4.3
betai=6.2
#Determination constant
f<-function(x){
  dgamma(x, 4.3,6.2)/dgamma(x,4,6)
}
C <- optimise(f,interval=c(0,10),maximum=TRUE)$objective

# Initialisation variables
X <- numeric(length = iterations) 
a <- logical(length = iterations)

for (r in (2:iterations)) {
  ## Obtain a proposed value
  proposal <- rgamma(n=1,alpha,rate=beta)
  
  ## Evaluate both densities at the proposed value
  target_density_value   <- dgamma(proposal,alphai,rate=betai)
  proposal_density_value <- C*dgamma(proposal,alpha,rate=beta)
  
  ## This is assured to be [0,1]
  acceptance_ratio <- (target_density_value / proposal_density_value)
  
  ## If U(0,1) is less than or equal to acceptance ratio, accept
  if(runif(n = 1) <= acceptance_ratio) {
    a[r] <- TRUE
    X[r] <- proposal
  }
  else{
    X[r]<-X[r-1]
  }
}

x=seq(0,3,0.01)

plot(x,dgamma(x,alphai,rate=betai),ylab="densité",xlab="x",type="h",col="red",main="densités des lois de probabilité générés")
lines(density(X))
legend(1,1,legend=c("denstité loi gamma sous R","densité obtenue avec l'Accept-Reject"),col=c("red","black"),lty=1)

On voit que les deux courbes de densité sont très proches (densité réelle en rouge et densité estimée par Accept-Reject en noir)

L'avantage de la méthode de Métropolis-Hastings est que l'on n'a pas besoin de définir une constante M (contrairement à l'Accept-Reject). #Repeat or Simulate

Repeat <- function(alpha, n){
  x <- runif(1)
  a <- 0
  for (i in 2:n){
    u <- runif(1)
    if (u > 1-x[i-1]){
      x [i]<- rbeta(1, alpha + 1, 1)
      a <- a+1
    }
    else{
      x[i]<- x[i-1]
    }
  }
  return(list("a" = a/n, "X" = x))
}

iterations=5000
mean=0
var=1


Metropolis <- function(delta){
  a <- logical(length = iterations)
  X=rep(runif(1),iterations)
  
  for (r in (2:iterations)) {
    ## Obtain a proposed value
    Y<-runif(n=1,min=-delta,max=delta)
    
    ## This is assured to be [0,1] et comme on a une loi uniforme avec le même intervalle (-delta-x;delta-x), on aura les mêmes valeurs de densité lorsque l'on applique la densité à x ou y (q(x|y)=q(y|x), q est symétrique), on peut limiter la proba à
    acceptance_ratio <- min(dnorm(Y,mean,var)/dnorm(X[r-1],mean,var),1)
    
    ## If U(0,1) is less than or equal to acceptance ratio, accept
    X[r]=X[r-1] + (Y-X[r-1])*(runif(1)<acceptance_ratio)
    a[r]=1*(runif(1)<acceptance_ratio)
  }
  quantile=quantile(X,0.9)-quantile(X,0.1)
  mylist<-list("X"=X,"a"=a,"quantile"=quantile)
  return(mylist)
}

Exercice 8.5¶

La fonction suit une loi de densité normale $$f(x) \propto exp{(-x^2/2)}$$.

En l'appliquant sur la première fonction du slice sampler, on trouve: $ w|x=U[0,f(x^{(t)})]=U[0, exp{(-x^2/2)}$]

Pour l'autre fonction de slice sampler, on doit résoudre en x:
$$f(x)\ge w \\ \Rightarrow exp{(-x^2/2)}\ge w \\ \Rightarrow (-x^2/2)\ge log{(w)} \\ \Rightarrow x^2/2\le -log{(w)} \\ \Rightarrow x^2\le -2 log{(w)} \\ \Rightarrow -\sqrt{(-2 log{(w)}} \le x \le \sqrt{(-2 log{(w)}} $$ On retrouve lors la deuxième fonction de l'énoncé: $ x|w=U[-\sqrt{(-2 log{(w)}}, \sqrt{(-2 log{(w)}}$]

On va maintenant implémenter l'algorithme de slice sampler pour ce cas de la loi normale (0,1).
