f<-function(x,d) (exp(-(x^d)))

finv<- function(x,d) (-log(x))^(1/d)

slicesampler <- function(init,d,iterations){
  #initialisation
  x=rep(init,iterations)
  u=rep(init,iterations)
  for(i in 2:iterations){
   u[i]=runif(1,min=0,max=f(x[i-1],d))
   x[i]=runif(1,min=0,max=finv(u[i],d))
  }
  return(x)
}

Exercice 7.22¶

A¶

Le problème du vendeur est de pouvoir passer voir tous ces clients et revenir à son point de départ en faisant le chemin le plus court possible afin de pouvoir le maximum de clients en limitant le temps de trajet pour le vendeur.

Le problème est le même, puisque la distance minimale est inchangée et elle reste minimale, si on fait une permutation circulaire des clients à voir et du point de départ. Ainsi le point de départ n'a pas d'importance.

Le problème peut alors être formulé comme pour N clients signalés par un numéro, qui représente l'ordre dans lequel ils vont être visités, une permutation $\sigma$ entre deux clients et une distance parcourue (ou à parcourir) entre le client i et le client j:

  • le vendeur part du point de départ 0
  • puis va voir ses clients 1, 2, 3, ...,N
  • puis il retourne au point de départ N+1=0
  • à chaque étape, on regarde la distance entre chaque client que l'on somme afin calculer la distance totale que l'on cherche à minimiser
B¶

On cherche à minimiser la distance totale parcourue par le vendeur en considérant $\sigma$ comme les permutations possibles après le client i. On cherche à minimiser: $$H(\sigma) = \sum\limits_{i=0}^N (i,\sigma (i)) $$ Dans notre cas, on cherche bien à minimiser ce problème et non celui de minimiser à chaque étape la distance, qui permettrait de trouver l'ensemble des minima locaux mais ne certifie en rien que l'on aura le minimum global.

C¶

Dans la partie d'initialisation:

  • repérer les clients (ici on va les repérer à l'aide de lois uniformes pour son abscisse et son ordonnée)
  • calculer la matrice de distance entre les clients
  • prendre de façon aléatoire un point de départ
  • calculer la distance totale
  • initialiser T

Dans la partie itérative:

  • prendre 2 nombre entre 1 et N
  • permuter l'ordre dans lequel le vendeur va voir ces clients représentés par leur nombre (qui sont les deux nombres choisis précédemment)
  • calculer la nouvelle distance totale
  • si la nouvelle distance est plus petite: on accepte cette permutation
  • si la nouvelle distance est plus grande: on l'accepte avec une probabilité $\exp{\delta/T}$ (Metropolis-Hastings)
  • mettre à jour T
D¶

On va maintenant implémenter ce problème, et voir la complexité du problème lorsque le nombre de clients augmente.

set.seed(2)
library(ggplot2)

Nombre_clients = 10
T = 4
#génération des coordonnées de chaque client
x = runif(Nombre_clients)
y = runif(Nombre_clients)
clients = data.frame(x)
clients$y = y
sigma = seq(from = 1, to = Nombre_clients)

dist_matrix = as.matrix(dist(cbind(clients$x, clients$y)))
dist_current = function(clients, dist_matrix) {
  distance = 0
    for (i in 1:Nombre_clients) {
      distance = distance + dist_matrix[match(i, sigma),match(i %% Nombre_clients + 1, sigma)]
    }
  return(distance)
}

dist_permut = function(clients, dist_matrix){
    distance = 0
    for (i in 1:Nombre_clients) {
      distance = distance + dist_matrix[match(i, sigma_prop),match(i %% Nombre_clients + 1, sigma_prop)]
    }
  return(distance)
}

prob_accept = function(a, b, T){
  exp(-(b - a)/T)
}

clients$trip1 = sigma %% Nombre_clients %/% 2
clients$trip2 = (sigma + 1) %% Nombre_clients %/% 2

ggplot(clients, aes(x = x, y = y, label = sigma)) +   geom_line(data = clients, aes(group = trip1)) +
  geom_line(data = clients, aes(group = trip2)) +  geom_point(size = 6, shape = 21, fill = "red") +
  geom_text() +  coord_fixed(xlim = c(0,1), ylim = c(0,1)) +
  ggtitle("Chemin initial")
In [23]:
d_actuel = dist_current(clients, dist_matrix)

iterations = 100000

best_dist = d_actuel
sigma_best = sigma
Dist<-c()
for (i in 1:iterations) {
  permut= sample(1:Nombre_clients, 2)
  
  sigma_prop = sigma
  sigma_prop[permut] = sigma_prop[rev(permut)]
  
  d_candidate = dist_permut(clients, dist_matrix)
  if (d_candidate <= d_actuel) {
    d_actuel = d_candidate
    sigma = sigma_prop
    if (d_actuel <= best_dist) {
      best_dist = d_actuel
      sigma_best = sigma
    }
  }
  else if (runif(1) < prob_accept(d_actuel, d_candidate, T)) {
    d_actuel = d_candidate
    sigma = sigma_prop
  }
  Dist[i]<-best_dist
  if (i %% 2500 == 0) {
    T = T * 0.7
  }
}

On voit que dans notre troisième cas, on n'a pas encore atteint le minimum puisque que l'on a encore des croisements de trajets. Alors que dans l'on a atteint le minimum dans les deux premiers cas, mais on va que ce problème n'a pas une complexité linéaire puisqu'en multipliant par trois le nombre de clients, on n'a pas encore atteint le minimum alors que l'on avait déjà à environ 2000 itérations, on aurait donc attendu qu'il converge à environ 6000 itérations, ce qui n'est pas le cas. Cependant, ce que l'on voit c'est que plus on a de clients plus on peut réduire le temps de trajet du vendeur de façon importante (dans le cas de 10 clients, on divise par deux le temps de trajet, dans le cas de 20 clients, on divise par 3 et dans le cas de 30 clients (alors que ce n'est pas optimisé) on a déjà divisé par plus de 3 le temps de départ).

Ici, on a un problème simplifié du vendeur puisque, dans la réalité, il y a également un problème sur le fait qu'il y ait une route de façon directe (en ligne droite) entre deux points consécutifs. Ici, on suppose que cette route existe toujours.
