f<-function(x,d) (exp(-(x^d)))

finv<- function(x,d) (-log(x))^(1/d)

slicesampler <- function(init,d,iterations){
  #initialisation
  x=rep(init,iterations)
  u=rep(init,iterations)
  for(i in 2:iterations){
   u[i]=runif(1,min=0,max=f(x[i-1],d))
   x[i]=runif(1,min=0,max=finv(u[i],d))
  }
  return(x)
}

Exercice 7.22¶

A¶

Le problme du vendeur est de pouvoir passer voir tous ces clients et revenir son point de dpart en faisant le chemin le plus court possible afin de pouvoir le maximum de clients en limitant le temps de trajet pour le vendeur.

Le problme est le mme, puisque la distance minimale est inchange et elle reste minimale, si on fait une permutation circulaire des clients voir et du point de dpart. Ainsi le point de dpart n'a pas d'importance.

Le problme peut alors tre formul comme pour N clients signals par un numro, qui reprsente l'ordre dans lequel ils vont tre visits, une permutation $\sigma$ entre deux clients et une distance parcourue (ou parcourir) entre le client i et le client j:

  • le vendeur part du point de dpart 0
  • puis va voir ses clients 1, 2, 3, ...,N
  • puis il retourne au point de dpart N+1=0
  • chaque tape, on regarde la distance entre chaque client que l'on somme afin calculer la distance totale que l'on cherche minimiser
B¶

On cherche minimiser la distance totale parcourue par le vendeur en considrant $\sigma$ comme les permutations possibles aprs le client i. On cherche minimiser: $$H(\sigma) = \sum\limits_{i=0}^N (i,\sigma (i)) $$ Dans notre cas, on cherche bien minimiser ce problme et non celui de minimiser chaque tape la distance, qui permettrait de trouver l'ensemble des minima locaux mais ne certifie en rien que l'on aura le minimum global.

C¶

Dans la partie d'initialisation:

  • reprer les clients (ici on va les reprer l'aide de lois uniformes pour son abscisse et son ordonne)
  • calculer la matrice de distance entre les clients
  • prendre de faon alatoire un point de dpart
  • calculer la distance totale
  • initialiser T

Dans la partie itrative:

  • prendre 2 nombre entre 1 et N
  • permuter l'ordre dans lequel le vendeur va voir ces clients reprsents par leur nombre (qui sont les deux nombres choisis prcdemment)
  • calculer la nouvelle distance totale
  • si la nouvelle distance est plus petite: on accepte cette permutation
  • si la nouvelle distance est plus grande: on l'accepte avec une probabilit $\exp{\delta/T}$ (Metropolis-Hastings)
  • mettre jour T
D¶

On va maintenant implmenter ce problme, et voir la complexit du problme lorsque le nombre de clients augmente.

set.seed(2)
library(ggplot2)

Nombre_clients = 10
T = 4
#gnration des coordonnes de chaque client
x = runif(Nombre_clients)
y = runif(Nombre_clients)
clients = data.frame(x)
clients$y = y
sigma = seq(from = 1, to = Nombre_clients)

dist_matrix = as.matrix(dist(cbind(clients$x, clients$y)))
dist_current = function(clients, dist_matrix) {
  distance = 0
    for (i in 1:Nombre_clients) {
      distance = distance + dist_matrix[match(i, sigma),match(i %% Nombre_clients + 1, sigma)]
    }
  return(distance)
}

dist_permut = function(clients, dist_matrix){
    distance = 0
    for (i in 1:Nombre_clients) {
      distance = distance + dist_matrix[match(i, sigma_prop),match(i %% Nombre_clients + 1, sigma_prop)]
    }
  return(distance)
}

prob_accept = function(a, b, T){
  exp(-(b - a)/T)
}

clients$trip1 = sigma %% Nombre_clients %/% 2
clients$trip2 = (sigma + 1) %% Nombre_clients %/% 2

ggplot(clients, aes(x = x, y = y, label = sigma)) +   geom_line(data = clients, aes(group = trip1)) +
  geom_line(data = clients, aes(group = trip2)) +  geom_point(size = 6, shape = 21, fill = "red") +
  geom_text() +  coord_fixed(xlim = c(0,1), ylim = c(0,1)) +
  ggtitle("Chemin initial")
In [23]:
d_actuel = dist_current(clients, dist_matrix)

iterations = 100000

best_dist = d_actuel
sigma_best = sigma
Dist<-c()
for (i in 1:iterations) {
  permut= sample(1:Nombre_clients, 2)
  
  sigma_prop = sigma
  sigma_prop[permut] = sigma_prop[rev(permut)]
  
  d_candidate = dist_permut(clients, dist_matrix)
  if (d_candidate <= d_actuel) {
    d_actuel = d_candidate
    sigma = sigma_prop
    if (d_actuel <= best_dist) {
      best_dist = d_actuel
      sigma_best = sigma
    }
  }
  else if (runif(1) < prob_accept(d_actuel, d_candidate, T)) {
    d_actuel = d_candidate
    sigma = sigma_prop
  }
  Dist[i]<-best_dist
  if (i %% 2500 == 0) {
    T = T * 0.7
  }
}

sigma = sigma_best

Alors que dans l'on a atteint le minimum dans les deux premiers cas, mais on va que ce problme n'a pas une complexit linaire puisqu'en multipliant par trois le nombre de clients, on n'a pas encore atteint le minimum alors que l'on avait dj environ 2000 itrations, on aurait donc attendu qu'il converge environ 6000 itrations, ce qui n'est pas le cas. Cependant, ce que l'on voit c'est que plus on a de clients plus on peut rduire le temps de trajet du vendeur de faon importante (dans le cas de 10 clients, on divise par deux le temps de trajet, dans le cas de 20 clients, on divise par 3 et dans le cas de 30 clients (alors que ce n'est pas optimis) on a dj divis par plus de 3 le temps de dpart).

Ici, on a un problme simplifi du vendeur puisque, dans la ralit, il y a galement un problme sur le fait qu'il y ait une route de faon directe (en ligne droite) entre deux points conscutifs. Ici, on suppose que cette route existe toujours.
